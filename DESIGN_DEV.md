## 1. 系统概述

### 1.1 核心功能
构建一个能够模拟"学生做题-对照标准-发现偏差-修正思维"完整学习闭环的智能体系统，通过多轮迭代优化生物学试题答案。

### 1.2 核心组件
1. **文档预处理**: 提取Word文档中的图片和文字，生成纯文本输入
2. **三智能体系统**: Actor (学生) → Reviewer (老师) → Reflector (反思者)
3. **状态管理**: 记录每轮迭代的完整历史和最终结果
4. **执行引擎**: 协调整个学习循环的执行流程

## 2. 已确认的技术架构

### 2.1 LLM策略 (混合方案)
| 组件 | 技术选择 | 理由 |
|------|----------|------|
| **图片处理** | 本地 qwen3-vl:8b (通过Ollama) | 大幅降低成本，本地模型处理图片生成描述 |
| **文本推理** | Deepseek API | 专业文本推理，成本可控 |
| **预处理策略** | Word→图片描述→纯文本输入 | 避免昂贵的多模态API调用 |

### 2.2 文档处理流程
```
Word文档 (.docx)
    ↓
python-docx 提取图片 + 文字
    ↓
Ollama (qwen3-vl:8b) 生成图片描述
    ↓
替换图片位置为描述文本
    ↓
纯文本输入到智能体系统
```

### 2.3 执行模型
- **完全串行执行**: 一道题一道题顺序处理
- **智能体内部串行**: Actor → Reviewer → Reflector 顺序执行
- **用户配置轮数**: 用户指定训练轮数，系统运行指定轮数后停止

### 2.4 数据持久化
- **格式**: JSON带版本控制
- **内容**: 每轮迭代的完整状态（输入、输出、评分、改进点）
- **目标**: 人类可读，便于调试，支持版本追踪

### 2.5 错误处理
- **策略**: 快速失败
- **原则**: 发生错误立即停止，需要手动干预
- **范围**: 不实现重试、降级和回退机制

## 6. 范围限制

### 6.1 明确不在范围内
1. **高级并行处理** - 保持完全串行执行
2. **复杂错误恢复** - 快速失败，不实现重试/降级
3. **性能优化** - 假设本地模型足够快，不实现缓存/批处理
4. **多模态API调用** - 预处理为纯文本
5. **复杂部署方案** - 本地Ollama运行
6. **质量阈值自动停止** - 固定轮数运行
7. **用户界面** - 命令行界面

### 6.2 技术假设
1. **本地模型性能**: qwen3-vl:8b在可用硬件上运行足够快
2. **API可用性**: Deepseek API稳定可用
3. **文档格式**: Word文档格式相对规范
4. **硬件要求**: 用户有运行本地模型的GPU/足够内存

## 9. 风险与缓解

### 9.1 技术风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 本地模型性能不足 | 处理速度过慢 | 使用量化模型，优化prompt |
| Deepseek API不稳定 | 系统不可用 | 快速失败，手动恢复 |
| 图片描述质量差 | 信息丢失 | 设计专门的生物学图片prompt |
| Word文档格式复杂 | 解析错误 | 逐步完善解析逻辑 |

### 9.2 项目风险
| 风险 | 影响 | 缓解措施 |

| 范围蔓延 | 项目延期 | 严格遵守"不在范围内"列表 |
